# Contributing to LSP D-PLANNER

Thank you for your interest in contributing to LSP D-PLANNER!

## How to Contribute

### Reporting Bugs
1. Check existing issues first
2. Create detailed bug report with:
   - Browser and OS
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots if applicable

### Suggesting Features
1. Describe the feature clearly
2. Explain the use case
3. Provide examples if possible
4. Note any potential impact

### Code Contributions
1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly
5. Submit a pull request

## Code Style

- Use Vanilla JavaScript (no frameworks)
- Keep functions focused and readable
- Add comments for complex logic
- Follow existing code patterns
- Test on multiple browsers

## Testing

- Test on Chrome, Firefox, Safari, Edge
- Test on iOS Safari and Android Chrome
- Test offline functionality
- Verify mobile responsiveness
- Check export formats (TXT, PDF, COPY)

## Safety

- Always emphasize safety disclaimers
- Never claim the tool replaces dive computers
- Include warnings about conservative planning
- Test all decompression calculations thoroughly

## Questions?

Open an issue or discussion on GitHub.
